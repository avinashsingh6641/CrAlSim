Understanding Advanced Encryption Standard (AES)

AES is a symmetric-key block cipher standardized by NIST in 2001. It processes data in 128-bit blocks using a 4x4 state matrix of bytes. The encryption is iterative, consisting of multiple rounds (10 rounds for 128-bit keys).

The Four Core Transformations per Round

Click on any transformation step below to view its visual explanation, mathematical background, and the actual JavaScript code used in this simulator.

Step 1

SubBytes

Non-linear byte substitution using a lookup table (S-Box).

Step 2

ShiftRows

Cyclic left shift permutation of the state matrix rows.

Step 3

MixColumns

Column-wise matrix multiplication over Galois Field GF(2^8).

Step 4

AddRoundKey

Bitwise XOR operation between the state matrix and the round key.

AES Round Structure

Initial Round (Round 0) AddRoundKey only
→
Standard Rounds (1 to 9) SubBytes, ShiftRows, MixColumns, AddRoundKey
→
Final Round (Round 10) SubBytes, ShiftRows, AddRoundKey (No MixColumns)

Input & Key Representation Input Phase

A 16-character string is converted to a 4x4 matrix of hexadecimal byte values called the State Array. The same is done for the 16-character secret key. We map characters column-by-column into the matrices.

Rounds 1 to 9: Standard Iterative Transformations Rounds 1-9

Each of these 9 rounds passes the state through 4 consecutive stages:

  1. SubBytes: Replace bytes using the S-Box. (e.g. 00 -> 63, 3c -> eb).
  2. ShiftRows: Cyclically shift Row 1 left by 1, Row 2 left by 2, Row 3 left by 3.
  3. MixColumns: Matrix multiplication over GF(2^8) to mix column values together.
  4. AddRoundKey: XOR with the Round Key derived via Key Expansion.

Round 10: Final Round (No MixColumns) Round 10

In the final round, we perform SubBytes, ShiftRows, and XORing with Round Key 10. We skip MixColumns to prevent it from neutralizing the diffusion gained in the previous rounds.

The resulting 16-byte state is read column-by-column to yield the final ciphertext.
