Understanding Data Encryption Standard (DES)

DES is a historical **symmetric-key block cipher** published by NIST in 1977. It operates on 64-bit blocks of plaintext using a 56-bit key, processing them through a **Feistel network** architecture consisting of 16 rounds of substitution and permutation.

Core Algorithm Flow

The encryption pipeline consists of several key stages:

  1. Initial Permutation (IP): Rearranges the 64 bits of the plaintext block in a fixed layout.
  2. Feistel Rounds: Splitting the block into Left (L) and Right (R) halves, and executing 16 rounds of:
    L_i = R_{i-1}
    R_i = L_{i-1} ⊕ F(R_{i-1}, K_i)
  3. The Round Function F: Expansion permutation (32 to 48 bits), Key XOR with round subkey, S-Box substitution (48 to 32 bits using 8 S-Boxes), and P-Box permutation.
  4. Final Permutation (IP⁻¹): The inverse of the initial permutation to produce the 64-bit ciphertext.

Key Schedule & Subkeys

The original key is 64 bits, but 8 bits are discarded (used as parity check bits), leaving a **56-bit active key**. This key is processed through circular left shifts and Permuted Choice tables to generate sixteen **48-bit subkeys** (K1 to K16), one for each Feistel round.

S-Box Byte Substitution

DES uses eight distinct **S-Boxes (Substitution Boxes)**. Each S-Box takes a 6-bit input and maps it to a 4-bit output. The outer two bits determine the S-Box row (0 to 3), and the inner four bits determine the column (0 to 15). This step provides the essential non-linear property of DES security.
