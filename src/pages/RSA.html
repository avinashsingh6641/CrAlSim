Understanding Rivest-Shamir-Adleman (RSA)

RSA is one of the first public-key cryptosystems and is widely used for secure data transmission. Unlike symmetric-key ciphers like AES, RSA is asymmetric, meaning it uses a pair of keys: a Public Key (shared with everyone for encryption) and a Private Key (kept secret for decryption).

Mathematical Foundation & Key Generation

The security of RSA relies on the practical difficulty of factoring the product of two large prime numbers (the factoring problem).

  1. Choose Primes: Select two distinct prime numbers P and Q.
  2. Compute modulus: Compute N = P * Q. N is used as the modulus for both keys.
  3. Compute Totient: Compute totient L = (P - 1) * (Q - 1).
  4. Choose E: Select an integer E (public exponent) such that 1 < E < L and E is coprime to L (i.e. gcd(E, L) = 1).
  5. Compute D: Compute D (private exponent) such that (D * E) % L = 1.

Keys Pair:
• Public Key = (E, N)
• Private Key = (D, N)

Encryption & Decryption Formulas

Encryption: Convert plaintext message M (as integer) to ciphertext C:
C = M^E % N

Decryption: Convert ciphertext C back to plaintext message M:
M = C^D % N

Step-by-Step RSA Example

1. Parameters Setup: Choose P = 11, Q = 13.

• N = 11 * 13 = 143.

• L = (11 - 1) * (13 - 1) = 10 * 12 = 120.

• Choose E = 7 (since gcd(7, 120) = 1).

• Find D where (D * 7) % 120 = 1 => D = 103 (since 103 * 7 = 721 = 6 * 120 + 1).

2. Encryption (Plaintext M = 9):

• C = 9^7 % 143 = 4782969 % 143 = 48.

3. Decryption (Ciphertext C = 48):

• M = 48^103 % 143 = 9.
